# 🌐 DNS / Fake-IP / Real-IP 说明



## 1. DNS 模式与体感差异

在目前的代理工具生态中，DNS 处理机制对网络体感（尤其是打开网页的瞬间响应速度）有显著影响。

* **Sing-box 模式**:
    * **评价**: 👑 最强 DNS 处理。
    * **体感**: 类似于 **120Hz 高刷屏**。无论是在 Real-IP 还是 Fake-IP 模式下，Sing-box 核心在 DNS 解析和缓存上的优化使其拥有目前“最快国内体感 + 最快国外体感”。
* **其他模式 (Mihomo/Xray/Surge 等)**:
    * **体感**: 类似于 **60Hz 标准屏**。虽然也很快，但对比 Sing-box 仍有微小差距。

## 2. Fake-IP vs Real-IP

* **Fake-IP (虚假 IP)**:
    * **原理**: 客户端不进行实际 DNS 解析，直接返回一个虚假 IP (如 198.18.x.x) 给应用，由远端节点解析。
    * **优点**: 响应极快，无需等待本地 DNS 返回。
    * **缺点**: 局域网设备互访、游戏联机、投屏等依赖真实 IP 的场景容易出问题（如 macOS 旁路网关方案中提到的“奇奇怪怪的问题”）。
* **Real-IP (真实 IP)**:
    * **原理**: 发起真实的 DNS 请求拿到真实 IP 后再连接。
    * **优点**: 兼容性好，适合游戏和局域网应用。
    * **缺点**: DNS 解析会消耗 RTT 时间，理论上比 Fake-IP 慢（除非使用 Sing-box 核心进行优化）。
